## Rebuilt Module 1 plan — Start with Confidence

### Module promise

The learner can enter the course without already knowing the type of computer they have, identify the operating system without guessing, distinguish safe exploration from risky action, and prepare a usable learning space.

### Lesson 1.1 — How This Course Works

#### Beginner explanation

- A computer course can show a screen, let the learner practice safely, and then guide the learner on the real device.
- `Lesson`, `practice`, `check`, `progress`, and `module` are explained with visible examples before they are used as course controls.
- Opening a lesson does not mean the skill is complete.
- The learner can reset a lesson or module without downloading a file.

#### Required visuals

1. Annotated course-player overview showing lesson navigation, practice, result check, progress, and reset.
2. Three-panel diagram: See it → Practice it → Do it on your computer.
3. Progress-state visual: Not started → Practicing → Demonstrated.

#### Required interaction

An orientation sandbox where the learner opens a sample lesson menu, selects a sample answer, resets the sample, and returns to the lesson. This does not alter real course progress.

#### Real-device practice

None required. Lesson 1.1 must work for a learner who does not yet have a computer available.

#### Recovery

The sandbox has a visible Reset sample button and cannot damage course progress.

### Lesson 1.2 — Identify the Computer and Operating System

#### Beginner explanation

- Laptop, desktop, all-in-one, and tablet/2-in-1 describe the physical device.
- Windows and macOS describe the main software that controls the computer.
- A laptop can run Windows; a MacBook is a laptop made by Apple and runs macOS.
- Case color, wallpaper, and browser do not reliably identify the operating system.

#### Required photography and screens

1. Windows laptop, desktop, and 2-in-1 as separate photographs.
2. MacBook and iMac as separate photographs.
3. Full Windows desktop screen with Start/taskbar context.
4. Close-up of Windows Start area.
5. Full macOS desktop with menu bar/Dock context.
6. Close-up of Apple menu and Finder/Dock clues.
7. Windows Settings verification screen.
8. macOS About This Mac verification screen.

#### Required interaction

A two-stage identification activity:

1. choose the physical device family from photographs;
2. choose Windows, Mac, or Neither/Unsure from screen evidence.

The course saves the operating-system path once. The choice screen provides Change my answer and Start this lesson over.

#### Real-device practice

The learner follows the saved Windows or Mac verification path and checks the operating-system name on the real computer.

#### Recovery

Neither/Unsure gives a visual verification path and never forces a guess. ChromeOS or another system receives an honest out-of-scope route.

### Lesson 1.3 — Green Light, Pause, or Stop

#### Beginner explanation

- Green Light: reversible exploration with low consequence.
- Pause: the learner needs to read, verify, or ask a question first.
- Stop: money, passwords, private information, permanent deletion, unknown downloads, physical danger, or unclear ownership is involved.
- A button's color does not determine the risk level; the consequence does.

#### Required visuals

1. Green/Pause/Stop decision diagram using icon, word, and color.
2. Visual examples: volume change, software permission, password request, payment, deletion, cable damage, and suspicious download.
3. Consequence path: Action → Possible result → Safe response.

#### Required interaction

Seven scenario cards classified by the learner as Green, Pause, or Stop. Feedback explains the consequence and recovery route. Scenarios must include at least one Windows example, one Mac example, one privacy example, and one physical-safety example.

#### Real-device practice

Perform one reversible Green Light action such as changing volume, then restore it.

#### Recovery

Every scenario includes What should I do now? and allows retry without penalty.

### Lesson 1.4 — Prepare a Safe, Comfortable Learning Space

#### Beginner explanation

- A stable, dry surface protects the computer.
- Vents need open space for airflow.
- Cables should not create a trip or pull hazard.
- Screen height, reach, brightness, and breaks affect comfort.
- The goal is improvement, not a perfect or expensive desk.

#### Required photography and diagrams

1. Real unsafe setup photograph.
2. Real improved version of the same setup.
3. Separate close-ups for blocked vent, liquid risk, stretched cable, unstable surface, glare, and unsupported reach.
4. Premium side-view posture diagram with adjustable alternatives for learners using different bodies, chairs, wheelchairs, or spaces.

#### Required interaction

A hotspot inspection where the learner finds hazards in the unsafe setup, receives feedback, and builds a safer arrangement from movable items.

#### Real-device practice

The learner improves at least two conditions around the real device and checks the result against visible criteria.

#### Recovery

The activity includes Reset room and explains safe alternatives when furniture or equipment cannot be changed.

## Module 1 mission — Start Without Guessing

The learner must:

1. operate the course sandbox and reset it;
2. identify the physical device family;
3. verify Windows, Mac, or an honest out-of-scope result;
4. classify four mixed-risk scenarios correctly;
5. identify hazards in a learning-space image;
6. perform and restore one safe real-device change.

Browser-verifiable steps are recorded automatically. Only the real-device change uses guided self-attestation, because the browser cannot inspect the operating-system setting.
